### Background

We as developer a lot of time and machine power in building our project. We test and build the project when we raise a PR.

We can save a lot of time and resources by speeding up our build process. In order to speed up we need to measure first. This is where build stats package come into play.

### Working of the Package

This package follows a simple principle, download all the data from source like, Bitbucket pipelines, Travis, or Circle CI and show in the tabular form for easy comparision.
